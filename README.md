# Compounding Intelligence Library

A library-themed personal site for the prompts you make and refine across the Compounding Intelligence course. Fork it, fill it with your own prompts, push to GitHub, and your library is live at `https://<your-username>.github.io/<this-repo>`.

Each prompt is a Markdown file in `prompts/`. The site reads those files, organizes them into shelves, and gives you one-click **Copy**, **Edit**, and **Open** for every one.

---

## Get your library running in 5 minutes

1. Click **Use this template** at the top of the GitHub page тЖТ **Create a new repository**. Name it anything (`my-reading-room`, `<yourname>-library`, etc.). Make it **public**.
2. In your new repo, open **Settings тЖТ Pages**. Under *Build and deployment* тЖТ *Source*, pick **GitHub Actions**.
3. Open **config.json** and change `owner`, `collection_name`, and `cohort` to yours. Commit.
4. Wait ~30 seconds. Your library is live at `https://<your-username>.github.io/<repo-name>`.

That's it. Every push to `main` rebuilds the site automatically.

---

## How to edit a prompt

Three ways, in order of speed.

**1. Edit in the browser, no commit.** Open your library, click a tile, click **Edit**. Your changes autosave to your browser's localStorage and persist across refreshes. Useful for iterating without polluting git history.

**2. Edit on GitHub.** From the open prompt view, click **Edit on GitHub**. It opens the underlying `.md` file in GitHub's web editor. Commit. The site rebuilds.

**3. Edit locally.** Clone the repo, open `prompts/*.md` in your editor, push. Same result.

## How to add a new prompt

Drop a new `.md` file into `prompts/`. Give it YAML frontmatter:

```
---
title: My New Prompt
shelf: Discovery
session: 2
lesson: 2
summary: One-sentence description that shows on the card.
date_filed: 2026-06-13
---

The body of the prompt goes here. Use fenced code blocks for the actual
prompt text тАФ the site shows a "Copy" button next to each one.
```

Push. The build script picks it up and it appears on the right shelf.

Available shelves (set the `shelf:` field to one of these): `Discovery`, `Research`, `Outreach`, `Building`, `Reflection`. Anything else goes to a "Misc" shelf.

## Running locally

You don't need to, but if you want to preview before pushing:

```
node scripts/build.js
python3 -m http.server 8000
```

Then open `http://localhost:8000`.
